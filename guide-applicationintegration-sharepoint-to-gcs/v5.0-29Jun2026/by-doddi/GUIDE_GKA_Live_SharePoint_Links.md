# 📖 Step-by-Step Guide: Configuring GKA Live SharePoint Citation Links (Phase 4a)

When contact center agents use **Google Cloud Generative Knowledge Assist (GKA)** or **Contact Center AI (CCAI)**, the AI assistant summarizes answers from the synchronized PDF reports in Google Cloud Storage (GCS). By default, clicking a source citation link attempts to open the raw storage blob (`gs://yourorg-bucket-sharepoint-sync/...`).

To ensure that agents clicking a citation link are immediately routed to the **live, interactive SharePoint web page** (`https://yourorg.sharepoint.com/...`), you must configure the frontend UI widget to extract custom metadata generated by our backend synchronization pipeline.

---

## 🏗️ How the End-to-End Architecture Works

1. **Backend Sync (Our Repository)**: During synchronization, our Traversal Cloud Function deposits a metadata file (`config/metadata.jsonl`) into GCS. Each record attaches a custom schema field called `sharepoint_url` to the underlying file.
2. **Vertex AI Ingestion**: Discovery Engine ingests the documents along with their `sharepoint_url` metadata attribute.
3. **Frontend UI Widget (Customer Portal)**: When displaying citations, the web UI inspects the document's metadata and overrides the hyperlink URL with the `sharepoint_url` attribute.

---

## 🛠️ Step-by-Step Implementation Guide for Frontend Developers

### Step 1: Locate the Agent Assist UI Module Configuration
In your web application or contact center portal codebase (e.g., `app.js`, `index.html`, or your CRM embedding script), locate the element or initialization script for `<agent-assist-ui-modules>`.

### Step 2: Inject `articleLinkConfig` into Knowledge Assist Settings
Add the `articleLinkConfig` property inside your Knowledge Assist module parameters. Set `linkMetadataKey` to `"sharepoint_url"` and `target` to `"_blank"` so links open cleanly in a new browser tab.

#### 📝 Example Implementation Snippet (JavaScript / HTML Web Component)

```html
<!-- Google Cloud Agent Assist UI Modules -->
<script src="https://www.gstatic.com/agent-assist-ui-modules/v1/container.js"></script>

<agent-assist-ui-modules>
  <knowledge-assist
    id="knowledge-assist-widget"
    project-id="YOUR_GCP_PROJECT_ID"
    location="global"
    knowledge-assist-config-id="YOUR_CONFIG_ID">
  </knowledge-assist>
</agent-assist-ui-modules>

<script>
  // Wait for the DOM and web components to initialize
  document.addEventListener('DOMContentLoaded', () => {
    const kaWidget = document.getElementById('knowledge-assist-widget');
    
    if (kaWidget) {
      // Configure citation links to use custom metadata 'sharepoint_url'
      kaWidget.config = {
        ...kaWidget.config,
        articleLinkConfig: {
          linkMetadataKey: "sharepoint_url",  // Matches key in config/metadata.jsonl
          target: "_blank"                    // Opens live intranet page in new tab
        }
      };
      console.log("✅ GKA Live SharePoint citation link configuration applied successfully.");
    }
  });
</script>
```

## 🧪 Verification & Testing Checklist

### Method A: Testing inside GCP Console (Agent Assist / Discovery Engine Simulator)
When testing inside the built-in **Google Cloud Console Simulator**:
1. Navigate to **Agent Assist > Simulator** (or **Discovery Engine > Preview**).
2. Ask a test query (e.g., *"What are the latest judging criteria?"*).
3. Inspect the returned citation metadata / JSON payload tab. You will confirm that the custom schema attribute `"sharepoint_url": "https://yourorg.sharepoint.com/..."` is cleanly attached to the citation record.
*(Note: Because the raw GCP Console Simulator is a generic administration tool, it does not execute custom frontend web component JavaScript overrides like `linkMetadataKey`. To test actual hyperlink click routing, use Method B below).*

### Method B: Testing inside Live Contact Center UI Widget
Once the frontend team embeds `<agent-assist-ui-modules>` with `articleLinkConfig`:
1. **Open the Agent Portal**: Navigate to the web CRM or portal where the widget is hosted.
2. **Ask a Test Question**: Ask a query triggering a SharePoint document citation.
3. **Inspect & Click Citation Hyperlink**: Hover over the citation chip. Confirm the browser status bar points directly to `https://yourorg.sharepoint.com/...`. Click the link to verify instant redirection to the live SharePoint intranet page via Entra ID Single Sign-On (SSO).

---

## 🆘 Troubleshooting

* **Link still opens raw GCS blob**: Ensure that Vertex AI Datastore indexing (Phase 4b) has run after the latest sync so that the `sharepoint_url` attribute is properly indexed in Discovery Engine.
* **404 or Access Denied in SharePoint**: Verify that the contact center agent logged into the browser has active Microsoft Entra ID permissions to view the target SharePoint subsite.
