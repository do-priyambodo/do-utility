**Add failure policy** ([Generally available (GA)](https://cloud.google.com/products?e=48754805&hl=en#product-launch-stages))

You can now configure more complicated [retry strategies for tasks](/application-integration/docs/error-handling-strategy), such as retries based on the error codes or the variable values during the execution:

+ Configure multiple ordered conditional failure policies for each task.
+ Configure a default failure policy that will be applied if no conditional failure policies matches.
+ Use [system auto-generated variables](/application-integration/docs/variables#system-variables) in the failure policies. For example, `ExecutionMode` and `ErrorInfo`.

For more information, see [Example for error handling](/application-integration/docs/error-handling#example).

**Local logging in async mode** ([Generally available (GA)](https://cloud.google.com/products#product-launch-stages))

By default, local logging for new integrations is now enabled in async mode. With this change, the log data is persisted (written) at fixed intervals or after the completion of the integration's execution, whichever is earlier. You can change the default settings by [editing the integration details](/application-integration/docs/view-integration-details#edit-integration-details).

For more information, see [Local logging](/application-integration/docs/local-logging).
