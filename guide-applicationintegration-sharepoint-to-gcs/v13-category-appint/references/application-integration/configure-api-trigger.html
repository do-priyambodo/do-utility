{{ apip_trigger_api }}s are used to directly invoke executions for integrations.

Configuring an {{ apip_trigger_api }} in your integration enables you to do the following:

  • Set request and response payload using trigger specific input and output variables.
  • Generate and view the OpenAPI Specification (OAS) for your integration.
  • Invoke an integration synchronously or asynchronously.
To help you get started with {{apip_trigger_api}}s, see the API fulfillment tutorial. The tutorial shows you how to create an integration with an {{apip_trigger_api}} that receives API requests for retrieving customer information.

Configure an {{ apip_trigger_api }}

To configure an {{ apip_trigger_api }}, click the {{ apip_trigger_api }} element in the integration editor. The {{ apip_trigger_api }} configuration pane appears showing the following fields:

  • Label: The visible name of the {{ apip_trigger_api }} in your integration. You can change the label to a name of your choice.
  • Notes: You can add notes to the {{ apip_trigger_api }} to provide additional information about the trigger.
  • Trigger ID: A unique ID that is assigned to the {{ apip_trigger_api }} in the format api_trigger/TRIGGER_NAME.
    Note: You can have more than one API trigger in your integration. Each API trigger is assigned a unique Trigger ID.
  • Input variables: Trigger specific input variables that you can use to set the request payload for the {{ apip_trigger_api }}.
    • To add or remove existing integration variables to the {{ apip_trigger_api }}, click Modify and select the required variables from the API trigger input variables pane.
    • To add a new variable, specifically for the selected {{ apip_trigger_api }}, click Modify and select + Add new variable from the API trigger input variables pane. For information about the supported data types, see Supported variable data types.
  • Output variables: Trigger specific output variables that you can use to set the response payload for the {{ apip_trigger_api }}.
    • To add or remove existing integration variables to the {{ apip_trigger_api }}, click Modify and select the required variables from the API trigger output variables pane.
    • To add a new variable, specifically for the selected {{ apip_trigger_api }}, click Modify and select + Add new variable from the API trigger output variables pane. For information about the supported data types, see Supported variable data types.
    • Note: You can create input and output variables from both the Variables pane and the {{ apip_trigger_api }} configuration pane.
    • {{integration}} automatically adds variables created from the {{ apip_trigger_api }} configuration pane to that specific trigger as input or output variables.
    • You can manually add variables from one {{apip_trigger_api}} to another as input or output variables.
    • By default, all elements in your integration can access variables created from the Variables pane.
    • Changing the Variable type of a variable affects all the elements in your integration that use that variable.

View OpenAPI Specification for your {{ apip_trigger_api }}

{{integration}} provides the ability to dynamically generate and view the OpenAPI Specifications of your published integrations that are configured with one or more {{ apip_trigger_api }}s. For information about how to view the OpenAPI Specification, see View OpenAPI Specification for your integration.

Note: This page focuses on invoking the API trigger using the v1 API. The OpenAPI Specification generated for your integration might include definitions for other API versions, such as v2.

Invoke an {{ apip_trigger_api }} using an API call

After you have published an integration that has an {{ apip_trigger_api }} configured, you can use the projects.locations.integrations.execute method to invoke that trigger from any application. This method triggers the execution in the synchronous mode.

Syntax

curl -X POST \
    -H "authorization: Bearer $(gcloud auth print-access-token)" \
    -H "Content-Type: application/json" \
    -H "X-Server-Timeout: TIMEOUT_SECONDS" \
    -d '{"trigger_id":"api_trigger/TRIGGER_NAME"}' \
    "https://LOCATION-integrations.googleapis.com/v1/projects/PROJECT_ID/locations/LOCATION/integrations/-:execute"

Replace the following:

  • TIMEOUT_SECONDS: The timeout in seconds for the API call. By default, the timeout is 120 seconds. For a list of pre-defined set of request parameters, see System parameters.
  • TRIGGER_NAME: The name of the {{ apip_trigger_api }} in your integration.
  • PROJECT_ID: The ID of your {{gcp_name}} project.
  • LOCATION: The location of your {{gcp_name}} project.

Example

The following example invokes an integration in us-east1 region containing the test_API_1 {{ apip_trigger_api }} in the demo-project Google Cloud project:

curl -X POST \
    -H "authorization: Bearer $(gcloud auth print-access-token)"
    -H "Content-Type: application/json" \
    -H "X-Server-Timeout: 300"
    -d '{"trigger_id":"api_trigger/test_API_1"}' \
    "https://us-east1-integrations.googleapis.com/v1/projects/demo-project/locations/us-central1/integrations/-:execute"

Considerations

When using the API trigger, the cumulative size of all the input variables can be a maximum of 8 MB. If the size exceeds 8 MB, {{shared_product_name}} doesn't run the integration and you will get an error.
