The {{ apip_task_whileloop }} task lets you make repeated calls to a {{sub_int}} from your current (main) integration until a specified condition is met.

The API Trigger ID and the Integration name identifies the {{sub_int}} you want to run.

The task also lets you collect the response from each {{ sub_int }} run and store the value in your current integration for use in downstream tasks. You can use any of the following approach to collect the responses:

  • Collate the response of each run in an array variable, where each element of the array has the response from one particular run. To collate the responses, configure the Collect values from {{ sub_int }} output(s) property.
  • Aggregate the responses of all the runs in a single variable. To aggregate the responses, configure the Override values from {{ sub_int }} output(s) property.
Note:
  • The {{apip_task_whileloop}} task can run a maximum of 8000 iterations. This constraint holds good even if the configured condition results in an infinte loop.
  • The cumulative size of the data collected from each {{ sub_int }} run cannot exceed 50 MB. If the size exceeds 50 MB, the integration fails, and the downstream tasks are not run.

Configuration properties

The following table describes the configuration properties of the {{apip_task_whileloop}} task.

Property Data type Description
API Trigger ID String Trigger ID of the {{ sub_int }} to run.

This value is available in the Trigger ID field of the API Trigger in the {{ sub_int }}.

Integration name String Name of the {{ sub_int }} to run.
While loop condition N/A Condition to terminate the loop. The {{ sub_int }} runs till the condition is met. For example, $loopMetadata.current_iteration_count$< 10.
Map to {{ sub_int }} input(s) N/A Values to pass from the current integration to the {{ sub_int }}.

The values of this property are in the form of key-value pairs, where the key is a variable in the current integration, and the value is the corresponding input variable in the {{ sub_int }}. For example, suppose your main integration has the productID variable, which you want to map to the subIntProductID {{ sub_int }} variable; you would select productID from the Integration variable to map from drop-down and enter subIntProductID in the Sub-Integration input to map to field.

Collect values from {{ sub_int }} output(s) N/A Collate the results of the {{ sub_int }} runs.

Every time the {{ sub_int }} runs, you can store the output of the {{ sub_int }} in an array variable of the current integration. Enter the {{ sub_int }}'s output variable in the Sub-Integration output to collect from field and select the array variable in current integration from the Integration variable to collect into drop-down. Each element of the array has the response from a particular {{ sub_int }} run. For example, if the array variable is resultsArray, resultsArray[0] has the response from the first run and resultsArray[1] has the response from the second run.

Override values from {{ sub_int }} output(s) N/A Read the output variable of a {{ sub_int }} run and store it in a variable of the current integration.

Select the {{ sub_int }}'s output variable from the Sub-integration output to map from drop-down, and select the corresponding variable in the current integration from the Integration variable to override drop-down. You can send the overridden variable in the current integration to the next {{ sub_int }} run by mapping the variable in the Map to additional {{ sub_int }} input(s) property. As a result, you can cumulatively aggregate the responses from the {{ sub_int }} runs.
Loop metadata JSON array The output variable that has the execution details of the {{ sub_int }} runs like the {{ sub_int }} execution ID, errors messages, and variable values passed during execution. Loop metadata contains variables including the following output variables: current_iteration_count, sub_integration_execution_ids, failure_location, current_element, iteration_element, and failure_message. {% include "application-integration/docs/_shared/_loop-metadata-warning.html" %}
